============================================================
  Agent Copilot App — Code Changes (2026-04-17)
  Fix: Missing checklist data for some interactions
============================================================

PROBLEM
-------
Some interactions that have checklist data in the Genesys Cloud
API Explorer were showing "No checklist" in the app. The root
cause was that API errors (429 rate limits, 5xx server errors,
network timeouts) during the background enrichment phase were
silently swallowed — the app treated them identically to "no
checklist data exists."

With 10 conversations enriched in parallel (each making 2–3 API
calls), 20–30 requests fire in rapid succession, which can
trigger Genesys Cloud rate limiting (HTTP 429).


CHANGES
-------

4 files changed, 75 insertions, 24 deletions.


1. js/services/apiClient.js — API retry with backoff
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Added automatic retry logic to the core API request function:
  - Retries up to 3 times on HTTP 429 (rate-limited) and 5xx
    (server error) responses
  - Respects the Retry-After header when present
  - Uses exponential backoff: 1s → 2s → 4s
  - Logs each retry to the browser console as a warning

BEFORE:
  A single failed API call would immediately throw an error.

AFTER:
  Rate-limited or server-errored calls are retried automatically
  before failing, significantly reducing silent data loss.


2. js/pages/dashboards/agent-copilot/agentChecklists.js — Error visibility
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
a) Error badge on failed rows:
   Rows where enrichment failed now show a red "⚠ Error" badge
   instead of "No checklist". Hovering the badge shows the actual
   error message (e.g. "API GET ... failed: 429 Too Many Requests").

b) Error count in status line:
   After enrichment completes, the status line now reports how many
   interactions failed, e.g.:
   "42 interactions — 38 with checklist data — 4 failed (hover badge
   for details)"

c) Better inner error handling:
   The per-communication checklist fetch now distinguishes genuine
   404s (no checklist exists — expected) from other errors (429, 500,
   network — logged as console warnings instead of debug messages).

d) Deduplication fix:
   The checklist deduplication filter previously dropped any checklist
   without an `id` field. Now falls back to the checklist `name` as
   the dedup key, preventing valid data from being silently discarded.


3. js/pages/dashboards/agent-copilot/checklistConfig.js — Missing media keys
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Added 3 missing communication media types to MEDIA_KEYS:
  - cobrowsesessions
  - screenshares
  - internalMessages

Without these, if an agent's communication was nested under one of
these keys, its communication ID would never be extracted and no
checklist API call would be made.

Added the "badgeError" label for the new error badge UI.


4. css/styles.css — Error badge styling
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
Added styling for the new .checklist-badge--error class:
  - Dark theme: red text (#f87171) on subtle red background
  - Light theme: red text (#dc2626) on subtle red background
  - cursor: help (indicates hoverable tooltip)
